3.1 Forking & Cloning a Remote Repository

Cloning a repository from GitHub means creating a copy of a remote repository on your local machine. This allows you to work on the project, make changes, and later push updates back to GitHub.

Why Clone a Repository?

  • Work Offline - You can make changes to a project without needing access to the internet
  • Collaboration - Teams can work simulteanously on the same project by cloning the same repository
  • Backup & Version Control - You can get a full version history of the project on your local machine
  • Save Time - You can clone starter code for projects instead of setting everything up manually

https://chedyhammami.medium.com/git-clone-vs-fork-in-github-610f158d61e3

Lesson Objectives

  • Understand the concept of forking a remote repository and how it allows you to create a personal copy for modification
  • Learn how to clone a remote repository to your local machine for making changes
  • Understand the difference between forking and cloning, and when to use each
  • Learn how to make changes in a cloned repository and push them back to the remote repository
  • Use git log to view the commit history and track changes in the repository

Let's start by cloning the training website so we can access it locally.

Step 1 of 4

Copy the GitHub Repository URL

  • Follow this link to the repository we're going to clone - Training Website
  • Click on the green button that says <> Code
  • Copy the link
Step 2 of 4

Clone The Repository

Now that we've copied the link to the website repository, we can use this to clone a copy of it on our local machines using Git Bash.

Note: Make sure you're in the git-exercises working directory in Git Bash. You can use the cd command to navigate out of your current directory.

Run the following command in Git Bash

git clone https://github.com/rockborne-git-training/git-github-website.git
Step 3 of 4

Find the Cloned Repository on Your Machine

  • In your File Explorer, search for your git-exercises folder
  • You should be able to see the cloned git-github-website repository
Step 4 of 4

Open the index.html File

Even though we won't be doing this on this repository, developers can now work collaboratively on the same code by pushing and pulling changes to and from the remote repository.

The next step for us is to open the website in a web browser so you can follow along with the rest of the course.

  • Double click on the git-github-website folder
  • Right click on the index.html file
  • Open the file with your preferred web browser

Lesson Summary

In this lesson, we learned how to clone a remote repository to our local machines using the git clone command. Cloning creates a copy of the repository on our local system, allowing us to work on it offline. We can then make changes, commit them, and later push the changes back to the remote repository for collaboration.

2.1 GitHub Setup and Creating a Remote Repository

What is GitHub?

GitHub is a cloud-based platform for hosting Git repositories. It allows developers to collaborate, track changes, and manage projects.

So far, we've created a local repository on our machines, but this is only accessible by us. To make the files available for everyone working simultaneously on a project, we need to push our files to a remote repository.

Let's take a look at how to do this.

Lesson Objectives

  • Create a GitHub account
  • Create a remote repository on GitHub
  • Understand when to initialise a repository with a README file

Creating a GitHub Account

Set up your GitHub account by following this link - GitHub Set Up

Step 2 of 2

Creating a Repository

  • Click on the ‘+’ symbol in the top right corner
  • Give your repository a name and a description
  • Set your repository to 'Private' for now as this is just an individual task
  • Do not initialise the repository with a README
  • Click 'Create repository'

Why wouldn't we initialise the repository with a README?

Whether or not to initialise a repository with a README file depends on the use case. Here's some reason why you might choose not to initialise a repository.

When to Initialise with README When NOT to Initialise with README
You are starting a new project from scratch You are pushing an existing project to the repo
You want the repo to be easily browsable on GitHub You are setting up a bare repository
The project is open-source and needs documentation from the start Your team prefers to add the README in the first meaningful commit
You want contributors to understand the project immediately The project structure requires manual file organization first

In our case, we want to push an existing project to the repository, so we won't initialise it with a README.

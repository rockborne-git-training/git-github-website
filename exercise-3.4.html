3.4 Git Log Continued

So far, you've cloned a repository onto your local machine and learned about the different git log commands we can use.

In this section of the exercise, you're going to be inserting text into the HTML file and create your own "About You" blog. You don't need any knowledge of web development to complete this exercise.

Then we'll use some of the git log commands to see your commit history.

Lesson Objectives

  • Use git log to view the commit history and track changes in the repository
  • Edit your own personal blog and track changes using commits and pushes
Step 1 of 12

Opening the Files in a Code Editor

You're going to be editing the code so that you can create a personal blog for yourself. This means that we need to open the HTML file in a code editor.

Path to find the index.html file: "C:\Users\YourName\git-exercises\git-exercise-02"

  • Right click on the index.html file
  • Open the file with your preferred code editor
Step 2 of 12

Insert Your Name

Insert your name where it says ‘Name’ inside the p tag on line 26 and save the file (Ctrl + S).

Step 3 of 12

Look At Your Changes in a Web Browser

Now that we've inserted our name on line 26, we should be able to see these changes in our website.

  • Navigate to your git-exercise-02 folder on your machine using File Explorer
  • Right click on the index.html file
  • Open with your preferred browser

Tip: Is your name not showing up? Make sure you've saved the file (Ctrl + S) after editing before refreshing the web page.

Step 4 of 12

git status

Remember, use git status as often as you can. It will help you to track your uncommitted changes and understand how the files are being tracked.

Now that we've made a change and running git status has told us the change hasn't been staged yet, let's go ahead and add it to the staging index.

Run the following command in Git Bash

git add .
Step 5 of 12

Committing The Change

With each change we make, even if it's small, we must still create a commit.

Run the following commands in Git Bash

git commit -m"Added first name"

If we need to be specifying multiple changes in the commit message, this is a sign that they should be part of multiple commits.

Step 7 of 12

Push the Change to GitHub

Now that we've modified files in our local repository, let's push this to the remote GitHub repository.

Run the following commands in Git Bash

git push origin master

Your GitHub repository should be updated with the most recent commit.

Step 8 of 12

Making Additional Changes

Let's go ahead and change more aspects of your blogs, makes more commits and push them with each new feature.

  • Go back to your code editor
  • Open the app.css file located in the explorer on the left
Step 9 of 12

Changing the Background Colour

To make it easier to visually see what changes we're making, go ahead and change the background to any colour of your choosing. You can either use a HEX code or, type in a simple colour.

  • In the app.css file, locate line 4 which says background-color
  • Insert any colour inbetween the colon and semi-colon
  • Save the change (Ctrl + S)
Step 10 of 12

Verify the Changes

It's good practice to make sure your changes are being implemented as you go along.

  • Go back to the web browser that has your blog and refresh the page
Step 11 of 12

Stage, Commit, Push

Now that we've modified another feature, it's time to stage the change, , then push it to your GitHub repository.

Run the following commands in Git Bash

git add .
git commit -m"Changed the background colour"
git push origin master
Step 12 of 12

Finishing The Blog

So far, you've learnt how to edit lines of code to insert your information, stage, commit and push your changes to your GitHub repository. Now it's time to finish the rest of your personal blog.

The sections left to fill out are your Job Role, Dream Holiday Destination, Favourite Hobby, Favourite Food and Favourite Drink. You can also add an image of yourself if you have one!

Now you can go ahead to this independently.

Tip: Make one change at a time, verify the change was successfully implemented, stage, commit and push the change.

You can go back through the lesson if you need a refresher of the steps.

Lesson Summary

In this lesson, you learned how to fork and clone a repository, bringing a copy of a remote project onto your local machine. You modified an HTML file, inserted personal details, and verified the changes in a web browser. Using git status, you tracked uncommitted changes before staging and committing them. You then pushed these updates to GitHub, ensuring your remote repository was up to date. This reinforced the importance of small, frequent commits for version control.

3.6 Branches

Branches are used to work on the same project, but in different isolated environments. By default, the first branch is named ‘master’, and this is the one we’ve also been working on.

In this part of the lesson, we're going to be looking at how to identify branches, create new branches, and delete branches.

https://www.atlassian.com/git/tutorials/using-branches

Why Use Branches?

  • Isolate features & bug fixes: Each branch is a seperate line of development. Developers can work on multiple features or bug fixes simultaneaously without interfering with the main branch
  • Safe experimentation: Developers can try new ideas in a branch without risking the stability of the main project. If the experiment doesn't work out, they can delete the branch without affecting the master branch
  • Parallel Development: Multiple developers can work on different branches at the same time and merge branches together after code reviews.

Let's take a look at how branches work

Step 1 of 5

Identify the Branch We're Currently On

Run the following command in Git Bash

git branch

This command will list all the branches in the repository.

The branch you're currently on will be denoted by an asterisk next to it and highlighted in green.

You can use this command to show the branch that is currently active.

Step 2 of 5

Creating a New Branch

Let's say we wanted to create a navigation bar feature for our project. We don't want to create this feature on the master branch in case something goes wrong, so we'll create a new branch and merge it into the master branch later.

Run the following command in Git Bash

git branch nav-bar

The git branch command will create a new branch and you specify the name of the new branch. In this case, we've called it nav-bar.

Step 3 of 5

Navigate Into the New Branch

Run the following command in Git Bash

git checkout nav-bar

The git checkout command is used to switch between branches

Step 4 of 5

Using git log to See What Branch We're On

When we run the following command, the HEAD should be pointing to nav-bar instead of master because we’ve switched branches.

Run the following command in Git Bash

git log --decorate
Step 5 of 5

Deleting Branches

What if we wanted to delete the branch we just created?

If we want to delete the branch we just made, we need to first switch branches, as you can’t delete a brunch you’re currently on.

Run the following commands in Git Bash

git checkout master
git branch -d nav-bar

We've checked out of the branch we want to delete and switched back to master. Then we used the -d flag to delete the nav-bar branch we just created.

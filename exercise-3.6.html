3.6 Branches

Branches are used to work on the same project, but in different isolated environments. By default, the first branch is named ‘master’, and this is the one we’ve also been working on.

In this part of the lesson, we're going to be looking at how to identify branches, create new branches, and delete branches.

https://www.atlassian.com/git/tutorials/using-branches

Why Use Branches?

  • Isolate features & bug fixes: Each branch is a seperate line of development. Developers can work on multiple features or bug fixes simultaneaously without interfering with the main branch
  • Safe experimentation: Developers can try new ideas in a branch without risking the stability of the main project. If the experiment doesn't work out, they can delete the branch without affecting the master branch
  • Parallel Development: Multiple developers can work on different branches at the same time and merge branches together after code reviews.

Let's take a look at how branches work

Step 1 of 10

Identify the Branch We're Currently On

Run the following command in Git Bash

git branch

This command will list all the branches in the repository.

The branch you're currently on will be denoted by an asterisk next to it and highlighted in green.

You can use this command to show the branch that is currently active.

Step 2 of 10

Making Additional Changes

Let's go ahead and change more aspects of your blogs on a different branch, makes more commits and push them with each new feature.

  • Go back to your code editor
  • Open the app.css file located in the explorer on the left
Step 3 of 10

Creating a New Branch

Let's say we wanted to change the background colour of our blog. We don't want to create this feature on the master branch in case something goes wrong, so we'll create a new branch.

Run the following command in Git Bash

git branch background-colour

The git branch command will create a new branch and you specify the name of the new branch. In this case, we've called it background-colour.

Step 4 of 10

Navigate Into the New Branch

Run the following command in Git Bash

git checkout background-colour

The git checkout command is used to switch between branches

Step 5 of 10

Using git log to See What Branch We're On

When we run the following command, the HEAD should be pointing to background-colour instead of master because we’ve switched branches.

Run the following command in Git Bash

git log --decorate
Step 6 of 10

Changing the Background Colour

To make it easier to visually see what changes we're making, go ahead and change the background to any colour of your choosing. You can either use a HEX code or, type in a simple colour.

  • In the app.css file, locate line 4 which says background-color
  • Insert any colour inbetween the colon and semi-colon
  • Save the change (Ctrl + S)
Step 7 of 10

Verify the Changes

It's good practice to make sure your changes are being implemented as you go along.

  • Go back to the web browser that has your blog and refresh the page
Step 8 of 10

Stage, Commit, Push

Now that we've modified another feature, it's time to stage the change, , then push it to your GitHub repository.

Run the following commands in Git Bash

git add .
git commit -m"Changed the background colour"
git push origin background-colour

Note: When we create a feature branch and make changes on it, typically, you would then merge it into the master branch. This is something we'll do in the next exercise.

Step 9 of 10

Deleting Branches

What if we wanted to delete the branch we just created?

If we want to delete the branch we just made, we need to first switch branches, as you can’t delete a brunch you’re currently on.

Run the following commands in Git Bash

git checkout master
git branch -D background-colour

We've checked out of the branch we want to delete and switched back to master. Then we used the -D flag to delete the background-colour branch we just created.

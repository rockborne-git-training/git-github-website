3.5 Tagging

In Git, tagging is a way to mark specific points in your repository's history, usually to denote important milestones like releases. Tags are similar to branches, but they don't move once created, they are fixed on the commit they reference.

Step 1 of 5

Looking at the Commit History

Let's look at the commits we've made so far.

Run the following command in Git Bash

git log --oneline
Step 2 of 5

Using the git tag Command

You may notice in Gat Bash that you haven't returned to the normal command line which starts with '$'. Instead, there's a colon. This is because when you run git log, Git opens the commit history inside a pager called less. This allows you to scroll through the output instead of flooding the terminal with too much information at once.

What are we placing a tag on?

Git tag will tag the most recent commit. If you want to tag a specific commit, you need to provide the SHA of the commit you want to tag e.g. git tag –a v1.0 369e8cf

Run the following command in Git Bash

git tag -a v1.0

What does this code mean?

  • git tag: The command used to create a new tag in Git.
  • -a: The flag that specifies the tag should be annotated, meaning it will include a message and metadata like the author and timestamp.
  • v1.0: The name of the tag being created, in this case, "v1.0" which could represent a version number or release.
  • This should open your code editor and will ask you to enter a message for the tag on line 1
  • Write "Ready for content"
  • Save the message (Ctrl + S) and close the code editor
Step 3 of 5

Verify the Tag

Run the following command in Git Bash

git tag

This should return the tag that was just created (v1.0).

Running this command will also list all tags created.

Step 4 of 5

Verify the tag Just Created was Associated With the Most Recent Commit

Run the following command in Git Bash

git log --decorate

The terminal should return: (HEAD -> master, tag: v1.0)

  • HEAD -> master: HEAD is pointing to the master branch
  • v1.0: A tag named v1.0 is pointing to the same commit
Step 5 of 5

Deleting Tags

What if we wanted to delete the tag?

Deleting a tag is useful if you accidentally mispelt something in the tag's message or if it was created by mistake.

Run the following command in Git Bash

git tag -d v1.0

The terminal should return a message saying the tag was deleted, along with the SHA of when the tag was created.

3.3 Git Log Commands

Git Log keeps track of every change made to a repository, and the git log command allows us to explore the history. You can find an old commit, see who made a change, and understand the development of a project using git log.

In this section, we'll cover different ways to use git log to efficiently navigate your project's history and find exactly what you need.

Different git log commands:

  • git log
  • git log --oneline
  • git log --decorate
  • git log --stat
  • git log -p
  • git log --author

Let's start exploring some of these.

Lesson Objectives

  • Use git log to view the commit history and track changes in the repository
  • Understand the different flags you can use with the git log command
Step 1 of 6

git log

This is the most basic git log command.

Run the following command in Git Bash

git log

As you can see, git log shows a list of commits in chronological order including the SHA, author, date of the commit and the message.

Step 2 of 6

git log --oneline

You may notice in Gat Bash that you haven't returned to the normal command line which starts with '$'. Instead, there's a colon. This is because when you run git log, Git opens the commit history inside a pager called less. This allows you to scroll through the output instead of flooding the terminal with too much information at once.

How to exit log mode?

Press q on your keyboard to quit and return to the normal command line.

Now let's explore the use of git log --oneline

Run the following command in Git Bash

git log --oneline

By running this command, we get the SHA of the commit and the message attached to it. This is useful when wanting a quick overview of commit history.

Step 3 of 6

git log --decorate

Now try running git log --decorate.

Run the following command in Git Bash

git log --decorate

You may notice that you get the same output as running git log.

This is because modern versions of Git often enable decoration by default. However, the key different is that git log --decorate explicitly ensures that branch names, tags and references are always displayed, even if they aren't shown by default in git log.

Step 4 of 6

git log --stat

Now try running git log --stat.

Run the following command in Git Bash

git log --stat

git log --stat will show the commit SHA, commit message and a summary of the file changes, inluding:

  • The name of each file modified
  • The number of lines added or deleted in each file
  • The summary of total changes in the commit

It's a quick way to see which files were modified and whether the changes were mostly additions or deletions.

Step 5 of 6

git log -p

Run the following command in Git Bash

git log -p

git log -p will show the full changes for each commit in the repository.

This means it will show exactly what lines were added, modified, or deleted for each commit.

After running log -p, we can see that one line of code was added in the last commit (highlighted in green).

  • Added lines are prefixed with a +.
  • Removed lines are prefixed with a -.
Step 6 of 6

git log --author

This command lets you filter out commits by specifying the author of a commit.

Run the following command in Git Bash

git log --author="Mary"

The author's name can be full or partial, or an email address, and Git will match commits based on this.

By default, this command is also case sensitive. You can make it case-insensitive by adding the --regexp-ignore-case flag after the command.

2.3 Pushing Changes to a Remote Repository

Pushing local repository changes to a remote repository ensures that your work is backed up, shared, and synchronised with others when you come to doing collaborative projects.

By pushing your changes to the remote repository, you create a cloud-based backup of your local work. This means that if something happens to your local machine, such as a crash or loss of data, you can easily retrieve the latest version of your work from the remote repository.

Now we're going to look at how we can push our exercise_1_questions.txt file which is currently in our local repository to our remote repository.

Step 1 of 2

Pushing Changes

We've already staged and committed any changes we've made to our Trivia file.

Without staging and committing, there's nothing to push because Git only tracks committed changes therefore, it's important that this process is completed before we try to push anything.

Run the following command in Git Bash

git push origin master
  • git push: Pushes the commit from your local repository to the remote repository
  • origin: The default name for the remote repository
  • master: The branch you're pushing changes to (we'll explore branches in later lessons)
Step 2 of 2

Verifying the Changes Were Pushed in GitHub

Your local Git repository should now be linked to the remote GitHub repository.

  • Head on over to your GitHub repository and refresh the page

What the GitHub Repository Reflects:

Code Changes: Any new commits that were pushed to the repository will appear in the commit history, showing the changes made, files modified, and the commit message

File Updates: If files were added, modified, or deleted, the updated files will appear in the file tree in the repository. New files or directories will show up, and deleted files will no longer be present.

Branches: If you pushed changes to a different branch, the branch will be listed in the "Branches" tab (we'll learn more about branches in later lessons)

Pull Requests: If your push was made to a branch that has an open pull request, the changes will be reflected in the PR immediately (we'll learn more about pull requests in later lessons)

Lesson Summary

In this lesson, we learned how to push changes to a GitHub repository. By using the git push command, we transfer local commits to the remote repository, updating it with our changes. After pushing, refreshing the repository shows new commits, file updates, and any changes to branches or pull requests. In the next lesson, we're going to look at cloning repositories.

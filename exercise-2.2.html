2.2 Linking a Local and Remote Repository

What is Linking a Repository?

Now that we've created a remote repository on GitHub, we need to link it to our local repository. Linking allows us to push our local changes to the remote repository and keep them synchronized for collaboration with others.

Let's walk through how to link the two repositories.

Lesson Objectives

  • Link a local repository to a remote GitHub repository
  • Verify that the remote GitHub repository was successfully linked
Step 1 of 2

Copy the GitHub Repository URL

  • Go to the GitHub repository you just created
  • It should prompt you with a link to set up your local repository with in the Quick setup section
  • Copy the link, and then open Git Bash again

Note: Make sure you're in the git-exercises\git-exercise-01 working directory in Git Bash

Run the following command in Git Bash

git remote add origin repository_url
Step 2 of 2

Verifying the Remote Repository was Added

Your local Git repository should now be linked to the remote GitHub repository.

It's good practice to verify that the intended actions were successfully run before we move on.

Run the following command in Git Bash

git remote -v

You'll see:

origin https://github.com/user/repo.git (fetch)

origin https://github.com/user/repo.git (push)

  • origin - Default name for the remote repository
  • fetch - URL used to pull updates
  • push - URL used to push changes

Lesson Summary

  • Learned how to link a local Git repository to a remote repository on GitHub
  • Copied the GitHub repository URL as the remote destination
  • Used git remote add origin to establish the link
  • Confirmed the connection with git remote -v

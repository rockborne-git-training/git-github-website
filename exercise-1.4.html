1.4 Modifying Commits

So far, we've made three commits to our local repository. The first was our initial commit, in the second commit we answered questions 1-3 on the .txt file and in the third commit, we answered questions 4 and 5.

What if we wanted to edit one of those previous commits?

Being able to edit a commit in Git is useful for maintaining a clear and accurate project history. It helps ensure that your commits are well-structured, meaningful and free from error.

To edit a specific commit in Git, you have a few options depending on whether it's the most recent commit or an earlier commit.

Lesson Objectives

  • Understand the importance of modifying and reverting Git commits to maintain a clean project history.
  • Learn how to use git commit --amend to modify the most recent commit's message.
  • Learn how to use git reset --hard to revert to a previous commit and restore file states.
  • Verify changes using git log to check the commit history after modifications or resets.
Step 1 of 4

Modifying the most recent commit

Let's imagine we made a mistake in the message of our most recent commit. We can use the git --amend command modify this.

Similar to the commit command, we can use the flag -m to specify a new commit message.

Run the following command in Git Bash

git commit --amend -m"Answered questions 4 and 5 on Trivia file"

By default, the ammend flag will edit the most recent commit.

Step 2 of 4

Verify The New Commit Message

Now that we've changed the commit message of our most recent commit, how can we verify that it was modified?

This is where the git log command comes in.

git log shows the commit history of a repository, allowing you to view past commits along with details like commit messages, authors, timestamps, and commit SHAs.

Run the following command in Git Bash

git log

As you can see, git log shows a list of commits in chronological order including the SHA, author, date of the commit and the message.

This is the most basic git log command. We can use additional flags with this command to retrieve specific information about a commit. We'll be taking a further look at these in later lessons.

Step 3 of 4

Reverting Back to a Previous Commit

Why would we want to revert to a previous file version?

  • Undoing Mistakes: Reverting to a previous version helps quickly undo errors or changes that break your code.
  • Recovering Lost Work: Reverting restores lost work if code is deleted or overwritten, as long as it was committed.
  • Version Control and Experimentation: Reverting allows safe experimentation, enabling you to undo mistakes without losing progress.
  • Bug Fixes and Rollbacks: Reverting lets you quickly roll back to a working version if a new feature or fix causes issues.

Let's take a look at how to go back to a previous version of your file.

After runing git log, we can see that every commit has a SHA which uniquely identifies that commit. We can use this SHA to target a specific commit we want to modify.

After running git log, copy the whole SHA of the commit where we answered the first three questions

Run the following command in Git Bash

git reset --hard commit-hash

What is this code doing?

  • Resets the commit history: Moves the current branch pointer to the specified commit, effectively "rewinding" the history.
  • Discards uncommitted changes: Removes all changes in the working directory and staging area that have not been committed.
  • Restores files to previous state: Resets all files in the working directory to match the specified commit.
Step 4 of 5

Verify the Reversion

Now that we've taken a look at some best practices for commit messages, let's create one ourselves.

Run the following command in Git Bash

git log

As you can see, this has reverted us back to a previous version of the file and has reset the commit history, removing the commit where we answered questions 4 and 5.

Step 5 of 5

Check the File

We can also verify that we reverted back to a previous version of the file by opening it in a text editor.

Run the following commands in Git Bash

notepad exercise_1_questions.txt

Your answers to questions 4 and 5 should now be gone as they weren't in the commit where we added the answers to questions 1-3, meaning we reverted successfully.

1.2 Staging a Change

In Git, staging a file means preparing a file to be committed. You will also see this stage be referred to as indexing, caching, or, staging area. But we'll be using the term staging throughout this course.

Think of staging as a preparation table for your files.

When you stage files, you're essentially marking the changes to be included in the next commit. Any files that aren't in the staging index won't be committed.

Step 2 of 4

Check the status of your directory

The git status command is important because it provides a snapshot of the current state of your Git repository. It helps you understand the following:

  • Which files are modified: It shows you which files have been changed but not yet staged for commit.
  • Which files are staged: It indicates which files are staged and ready to be committed to the repository.
  • Untracked files: It displays any new files that aren't yet being tracked by Git, so you can decide whether to add them to version control.

Run the following command in Git Bash

git status

When you run git status and see "Untracked files", it means that Git has detected files in your working directory that are not currently being tracked by version control.

They are not part of any commit and won't be included in future commits unless you tell Git to track them.

Step 3 of 4

Staging Changes

Staging changes in Git is the process of selecting specific changes in your working directory and preparing them to be included in the next commit. Using the git add command, you can stage individual files or all changes at once.

Run the following command in Git Bash

git add exercise_1_questions.txt

Note: You can put a full stop after git add to specify you want all files with modifactions made to them to be tracked, rather than listing them individually
